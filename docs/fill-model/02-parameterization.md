# 02 — c(x) 参数化形式与模型选择

## 1. 单段指数（Phase 1 基线）

```
c(x) = ten_b × exp(k × x),  k < 0
```

- 参数少（2个/侧），稳定，可解释
- `log c(x) = log(ten_b) + k·x` — 线性
- 适用于 log ĉ(x) vs x 近似线性的情况

C++ `TailFillModel` 存储格式（6 列）：
```
slope, intercept, end, ten_b, k, prefix_end
```
其中 `prefix_end = ∫_0^end c(x)dx = ten_b × (exp(k×end) − 1) / k`

## 2. 分段指数（Phase 1 推荐）

```
c(x) = ten_b_j × exp(k_j × x),  x ∈ [b_j, b_{j+1})
```

**分段边界**（推荐）：
```
[0, 50, 100, 200, 500, 1000, 2000, 5000, 10000] USD
```

优势：
- 浅/深档可用不同的 `k`
- 直接映射到 `TailFillModel` 的 `pp` 格式
- 支持 BID/ASK 独立分段

Phase 0 结果显示浅档线性好但深档有偏离 → 分段指数是正确选择。

## 3. 指数+幂律混合（Phase 4 目标）

```
log c_s(x,z) = a_s + k_s·x + β_s·log(x+x₀) + θ_s^⊤·z + η_s^⊤·(z·x)
```

- 同时覆盖指数衰减（`k_s·x`）和重尾（`β_s·log`）
- 支持 regime 条件化（`z` 项和交互项）
- **当前 C++ `TailFillModel` 不支持此形式** — 需扩展或离线拟合后压缩成分段近似

## 4. 第一版路径选择

| 路径 | 做法 | 阶段 |
|------|------|------|
| **A. 纯分段指数**（推荐） | Phase 1 输出分段指数 CSV，接现有 C++ | Phase 1 |
| B. 混合模型→分段近似 | 离线拟合混合模型，再压成分段指数 | Phase 4 |
| C. 扩展 TailFillModel | C++ 支持混合形式 + regime 参数 | Phase 4 |

**第一版走路径 A**。

## 5. 模型选择标准

不预设形状，用数据决定：

| 诊断图 | 结论 | 模型 |
|--------|------|------|
| `log c(x) vs x` 线性 | 单段指数够 | `ten_b × exp(kx)` |
| `log c(x) vs x` 弯曲 | 分段或混合 | 分段指数 × N |
| `log c(x) vs log x` 线性 | 幂律更合适 | `x^{−α}` |

## 6. 参数的经济含义

| 参数 | 含义 | 大值 → | 小值 → |
|------|------|--------|--------|
| `ten_b` | 浅档到达率 | 活跃市场 | 冷清市场 |
| `|k|` | 深度衰减速度 | fills集中浅档 | fills分散深档 |
| `β`（Phase 4） | 重尾程度 | 深档仍有fills | 深档fills快速归零 |

Phase 0 实证：
- USDT `|k|~0.00005`：$10000深档仍保留65% fills → 可以挂深档赚spread
- EUR `|k|~0.0008`：$1000深档只剩44% fills → 必须挂浅档快进快出
